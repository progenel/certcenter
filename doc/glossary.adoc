== Термины и технологии

* **CA (Certification Authority)** — центр сертификации, подписывает сертификаты X.509.
* **Root CA** — корневой офлайн-центр сертификации, которому доверяют все цепочки.
* **Intermediate CA** — онлайн-центр сертификации, подписывает пользовательские сертификаты от имени root.
* **X.509 сертификат** — электронный документ, содержащий открытый ключ, данные владельца и подпись CA.
* **CSR (Certificate Signing Request)** — запрос на выпуск сертификата, включает открытый ключ и данные владельца, подписан приватным ключом заявителя.
* **CRL (Certificate Revocation List)** — список отозванных сертификатов, подписан CA, публикуется для проверки статуса.
* **OCSP (Online Certificate Status Protocol)** — онлайн-проверка статуса сертификата (good/revoked/unknown) через ответ сервера OCSP.
* **Цепочка сертификатов (chain)** — набор сертификатов (intermediate + root) в одном файле PEM для валидации доверия.
* **PKCS#11** — стандартный интерфейс к криптоустройствам/HSM/soft-HSM (операции с ключами, подпись, шифрование).
* **Soft-HSM** — программное хранилище, реализующее интерфейс PKCS#11 без аппаратного модуля.
* **PKCS#12 (P12/PFX)** — контейнер с сертификатом и закрытым ключом, защищенный паролем.
* **Envelope-шифрование** — схема, где файл шифруется симметричным ключом (AES), а сам ключ шифруется открытыми ключами получателей.
* **AES-GCM** — алгоритм симметричного шифрования с аутентификацией (обеспечивает целостность и конфиденциальность), относится к AEAD.
* **AEAD (Authenticated Encryption with Associated Data)** — режимы шифрования, одновременно обеспечивающие конфиденциальность и целостность (пример: AES-GCM).
* **IV (Initialization Vector)** — инициализационный вектор, случайное значение для уникализации шифрования; не секретен, но должен быть уникален, хранится вместе с шифротекстом.
* **Nonce** — одноразовое значение, гарантирующее уникальность операции (часто совпадает по назначению с IV в AEAD-режимах).
* **Salt** — случайное значение для ключевых derivation функций (KDF), защищает от радужных таблиц и переиспользования ключей; применяется в хешировании паролей и KDF.
* **RSA/ECC** — пары открытый/закрытый ключ для шифрования/подписи; RSA 2048/3072 или эллиптические кривые (P-256).
* **LDAP/AD** — каталог пользователей (например, Active Directory) для аутентификации и получения атрибутов.
* **OpenVPN** — VPN-сервер/клиент, использующий X.509 сертификаты для аутентификации.
* **S3/minio** — объектное хранилище; используем для CRL, цепочек, пакетов (VPN/p12, envelope-архивы).
* **Redis** — память-кэш и очередь для фоновых задач (обновление CRL, уведомления).
* **SQLite/PostgreSQL** — базы данных: SQLite для MVP (минимум зависимостей), PostgreSQL — целевая.
* **FastAPI** — web-фреймворк на Python для API.
* **Pydantic** — типизированные модели данных для валидации/сериализации в FastAPI.
* **OpenAPI/Swagger** — спецификация и UI для описания и тестирования REST API.
* **Фабрика приложения (factory)** — функция, которая создает и возвращает сконфигурированный экземпляр приложения (например, FastAPI), позволяя запускать несколько экземпляров с разными настройками и упрощая тестирование.
* **UI (User Interface)** — пользовательский интерфейс (веб-портал).
* **CLI (Command Line Interface)** — интерфейс командной строки.
* **UX (User Experience)** — пользовательский опыт; удобство и понятность интерфейса/потоков.
* **RBAC** — разграничение доступа по ролям (администратор, офицер безопасности, пользователь, аудитор).
* **2FA** — двухфакторная аутентификация (дополнительный фактор при входе администраторов).
* **C4 (Context/Container/Component/Code)** — метод моделирования архитектуры; здесь используем уровень контейнеров (UI/CLI, API, CA core, Crypto, Storage).
* **Container (C4)** — логический/развернутый блок системы (приложение или сервис), взаимодействующий по четким интерфейсам.
* **EKU (Extended Key Usage)** — расширенные цели использования сертификата (например, clientAuth, emailProtection).
* **TTL (Time To Live)** — срок действия/жизни объекта, для сертификатов — период notBefore/notAfter.
* **FR (Functional Requirements)** — функциональные требования к системе (что она должна делать).
* **NFR (Non-Functional Requirements)** — нефункциональные требования (производительность, безопасность, масштабируемость и др.).
* **Backlog** — приоритезированный список задач/требований (обычно user stories), из которого выбирают работы для очередных итераций/MVP.
* **SLA/SLO** — метрики доступности и целевые показатели сервиса.
* **RPO (Recovery Point Objective)** — допустимая потеря данных при сбое.
* **RTO (Recovery Time Objective)** — целевое время восстановления работы после сбоя.
* **HA (High Availability)** — отказоустойчивость, обеспечение минимального простоя.
* **ABAC (Attribute-Based Access Control)** — управление доступом на основе атрибутов (альтернатива/расширение RBAC).
* **Key Escrow/Recovery** — политика восстановления ключей (если допускается бизнесом).
* **TLS** — протокол защиты транспорта; требования к версиям/шифрам (например, TLS 1.2+).
* **CMK/KEK/DEK** — уровни ключей (мастер, оборачивающий, данных) в схемах шифрования.
* **Idempotency** — свойство операций выполнять одно и то же действие при повторе без побочных эффектов (важно для API).
* **IdP/SP, SSO, OIDC/SAML** — поставщик удостоверений/сервис, единый вход, протоколы внешней аутентификации.
* **Non-repudiation (неотказуемость)** — невозможность оспорить факт подписи/операции.
* **API versioning** — управление версиями API для совместимости.
* **Rate limiting/Throttling** — ограничение числа запросов, защита от перегрузки.
* **Backpressure/Очереди** — управление нагрузкой через очереди/отложенные задачи.
* **Observability** — наблюдаемость (метрики, логи, трассировки).
* **CI/CD** — непрерывная интеграция/доставка.

* **DN (Distinguished Name)** — уникальное имя субъекта сертификата, набор полей вроде CN, O, OU, C.
* **CN (Common Name)** — имя субъекта (часто FQDN или имя пользователя).
* **O (Organization)** — организация/компания.
* **OU (Organizational Unit)** — подразделение или роль.
* **C (Country)** — страна.
* **Срок действия (Validity)** — период действия сертификата (notBefore/notAfter), часто называют сроком жизни сертификата.
* **Политики DN** — правила для Distinguished Name: допустимые поля (CN/O/OU и др.), форматы и ограничения, которые применяются при выпуске сертификатов.
* **Администратор** — управляет пользователями, ролями, интеграциями (LDAP/AD, S3/minio, Redis, уведомления), базовыми настройками CA.
* **Офицер безопасности / криптоофицер** — управляет профилями сертификатов (EKU, сроки, DN), выпускает/отзывает/продлевает сертификаты, контролирует CRL/OCSP, отслеживает аудит криптоопераций.
* **Пользователь / Сотрудник** — запрашивает сертификаты (CSR или серверная генерация), скачивает chain/CRL, шифрует/расшифровывает файлы, использует cert для подписи, шифрования и VPN.
* **Аудитор** — просматривает журналы аудита (выпуск/отзыв/доступ/шифрование), проверяет доступность CRL/chain.
* **МСП (малые и средние предприятия)** — организации с ограниченными ресурсами, требующие простых и недорогих решений.
* **ЭДО (электронный документооборот)** — обмен и хранение документов в электронной форме, включая подпись и проверку сертификатов.
* **Workflow** — бизнес-процесс/маршрут согласования (пошаговое выполнение задач и проверок, включая подпись/проверку статуса cert).
* **CAdES/XAdES** — форматы криптографических подписей для ЭДО (CMS/XML с поддержкой цепочек, статуса сертификатов, меток времени).
* **Артефакты** — результаты сборки/выпуска, отличные от исходного кода (документы, диаграммы, пакеты сертификатов/профилей, дистрибутивы, отчеты, бинарные файлы); хранятся отдельно от исходников. Ссылки (например, S3 URL) и метаданные (версии, контрольные суммы) фиксируются в БД.
* **Радужные таблицы** — заранее вычисленные таблицы для обратного поиска паролей/хешей; использование salt делает их бесполезными.
